## ⚠️ 중요 알림: CareFlow v4.1 🚦 신호등 × "Related To" 프레임워크

**CareFlow는 의학적 진단을 하지 않습니다.**
주관적(S) + 객관적(O) 자료를 통합하여 질병과 관련된 심리적/신체적 현상을 파악하고,
자기돌봄을 지원하는 간호학적 도구입니다. 의료와 협력하여 건강 리터러시를 증진합니다.

### 🚦 신호등 × NANDA 분류

| 🚦 | NANDA 표현 | 의미 | CareFlow 응답 |
|----|-----------|------|--------------|
| 🟢 | Readiness for enhanced ~~ | 건강, 향상 의지 | 자기돌봄 강화 |
| 🟡 | Risk for ~~ | 위험 신호 감지 | 예방 모니터링 |
| 🟠 | Actual ~~ Related To | 현상 발생 | 현상 기반 지원 |
| 🔴 | Potential Complication of ~~ | 의료 필수 | 즉시 의료 연결 |

### v4.1 접근 방식 (Related To + 신호등)

**v4.0 표현 (❌ 진단처럼 들림)**
```
CareFlow: "이건 우울증입니다"
```

**v4.1 표현 (✅ 신호등 + 현상 파악)**
```
S(주관): "2주간 아무것도 안 하고 싶어요"
O(객관): 활동량 70% 감소, 수면 증가

🔴 협력문제 감지: PC: Major depression → 정신과 연결
🟠 현상 파악: "무력감과 관련된 활동 감소를 경험하고 계신 거군요"

CareFlow: "정신과 상담이 필요할 것 같아요.
동시에 🟠 자기돌봄을 함께 시작해 봐요."
```

**그 질병에 대한 인간의 반응(심리적 영향, 불안감, 생활 제약)에만 집중합니다.**

**신체 증상과 정신 증상을 동등하게 취급합니다:**

신체 증상 영역:
- ❌ 이명, 비문증, 어지럼증 진단
- ❌ 메니에르병, 편두통 진단

정신건강 영역:
- ❌ 우울증, 불안장애, 공황장애 진단
- ❌ 공황장애, PTSD 진단
- ❌ 정신증상(환청, 환각) 진단

공통 금지 사항:
- ❌ 약물 처방 또는 의학적 조언
- ❌ 의료 진단명 제시

허용되는 CareFlow 역할:
- ✅ 증상으로 인한 심리적 어려움을 함께 관찰
- ✅ 일상에서의 자기돌봄 방법 제시
- ✅ 신체/정신 건강 전문가 상담 권유

**심각한 정신건강 문제나 신체 증상이 있으시면 반드시 전문의와 상담하세요.**

## 🧠 4축 증상 감지 엔진 설계 (Symptom Detection Engine)

> 사용자의 채팅 입력에서 4개 축(몸·감정·관계·의미)을 자동으로 감지하고 점수화하는 엔진입니다.

### 1. 데이터 분류 체계 (Taxonomy)

| 축 | 핵심 키워드 | 점수 |
|----|------------|------|
| 몸 (Body) | 이명, 어지러움, 피로, 두통, 윙윙, 빙글, 무겁다 | 각 1점 (최대 4점) |
| 감정 (Emotion) | 불안, 긴장, 예민, 짜증, 두려움, 무섭다, 떨린다 | 각 1점 (최대 4점) |
| 관계 (Relation) | 사람, 대화, 소통, 고립, 혼자, 친구, 가족, 모임 | 각 1점 (최대 3점) |
| 의미 (Meaning) | 성취, 의미, 계획, 보람, 성공, 해내다, 목표 | 각 1점 (최대 3점) |

총 일일 최대 점수: **14점** (몸4 + 감정4 + 관계3 + 의미3)

---

### 2. 감지 로직 (Hybrid Detection Logic)

사용자 채팅 입력 시 다음 순서로 엔진이 동작합니다.

#### Step 1 — 키워드 스캐닝

입력된 문장에서 위 표의 단어가 포함되었는지 전수 조사합니다.

```
예: "오늘 이명 때문에 사람 만나기 싫었어"
→ [몸] 감지 (이명), [관계] 감지 (사람)
```

#### Step 2 — 문맥 분석 (Apple Natural Language, On-Device NLP)

- **NLLanguageRecognizer**: 사용 언어 확인 (한국어)
- **NLTagger**: 문장 감성(Sentiment) 분석
  - 부정(-) → 해당 축 점수 기록
  - 긍정(+) → '의미' 또는 회복 지표에 반영

모든 NLP 처리는 기기 내부(On-Device)에서 실행되며, 원문 텍스트는 서버로 전송되지 않습니다.

#### Step 3 — 부족한 축 질문 유도 (Feedback Loop)

4개 축 중 감지되지 않은 항목이 있을 경우 챗봇이 자연스럽게 질문을 유도합니다.

```
예: 몸·관계만 감지된 경우
챗봇: "오늘 감정 상태나 하루의 보람(의미)은 어떠셨나요?"
```

---

### 3. 기능 명세 (Feature Spec)

#### 실시간 시각화 (Real-time Mirroring)

- 사용자가 채팅을 한 줄 입력할 때마다 상단의 4축 대시보드가 실시간으로 업데이트됨
- 표시 원칙: "상태가 나쁘다" 표현 금지 → **"기록이 업데이트되었습니다"** 로만 표시

#### 데이터 익명화 및 로컬 저장

- 환자 데이터이므로 서버로 전송하지 않고 iPhone 내부에만 저장 (**CoreData** 활용)
- 의료법 및 개인정보 보호법상 강력한 장점
- 분석은 전적으로 On-Device에서 수행 (Apple Natural Language 프레임워크)

#### 점수 집계 방식

```
일일 점수  = 해당 날 감지된 각 축 점수 합산
주간 점수  = Σ(일일 점수) ÷ 기록한 날 수
월간 점수  = Σ(일일 점수) ÷ 기록한 날 수
```

동일 키워드가 같은 날 여러 번 등장해도 해당 축 점수는 최대값을 초과하지 않습니다.

## 📡 데이터 수집 엔진 설계 (Data Collection Engine)

> iPhone·Apple Watch의 표준 API만 사용하여 투명하게 데이터를 수집합니다. 모든 처리는 On-Device에서 이루어집니다.

---

### 1. 주변 소음 및 음성 레벨 기록

> "나의 목소리와 주변 환경의 조화를 기록합니다."

iPhone 내장 마이크(AVFoundation)를 통해 주변 소음 레벨을 실시간으로 확인합니다. 환경 소음이 지나치게 높거나 내 목소리가 설정된 기준보다 올라갈 경우, 햅틱 진동으로 알림을 주어 편안한 대화 톤을 유지하도록 돕습니다.

- **수집 API**: `AVAudioSession` + `AVAudioEngine` (AVFoundation)
- **알림 조건**: 환경 소음 ≥ 기준값 또는 음성 레벨이 평소 대비 일정 dB 초과 시
- **알림 방식**: 햅틱 피드백 (`CoreHaptics`) — 진단적 판단 없이 단순 알림만 제공
- **저장**: 발생 시각 + dB 수치만 CoreData에 로컬 저장, 서버 미전송

---

### 2. 보행 및 활동 데이터 매칭

> "증상의 순간, 건강 데이터와 나란히 비교해 보세요."

Apple HealthKit의 보행 데이터를 활용합니다. 어지러움 기록 시점의 보행 안정성 추이와 걸음 비대칭성 데이터를 타임라인에 나란히 배치합니다.

**미러링 원칙**: 앱의 독자적 판단 없이 "당시 기록된 Apple 건강 데이터"를 그대로 보여주는 방식으로만 표시합니다.

> ※ 그래프 하단 캡션: *본 데이터는 Apple 건강 앱에서 제공된 수치입니다.*

- **수집 항목**: 보행 안정성(Walking Steadiness), 걸음 비대칭(Walking Asymmetry), 보행속도(Walking Speed), 보폭(Step Length), 이중지지시간(Double Support Time)
- **수집 API**: `HKHealthStore` (HealthKit)
- **저장**: HealthKit 원본 데이터를 직접 참조, 별도 복사본 미생성

---

### 3. 연동 데이터 소스 (Data Sources)

표준 API를 통한 투명한 데이터 수집:

| 데이터 항목 | API | 비고 |
|-----------|-----|------|
| 심박수 및 활동량 | Apple HealthKit | 자동 연동 |
| 보행 안정성 및 비대칭 기록 | Apple HealthKit 이동성 데이터 | 미러링 방식 |
| 주변 소음 및 음성 레벨 | AVFoundation | On-Device 처리 |

---

### 4. 일상 기록 통합 시각화

> "흩어져 있던 기록들이 하나의 일기가 됩니다."

수집된 데이터는 **4축(몸)** 지표에 자동으로 기록됩니다. 발생 시간, 주변 환경, 복약 여부를 통합하여 주간·월간 단위의 대시보드로 시각화합니다. 이를 통해 사용자는 자신의 일상 흐름을 객관적으로 살펴볼 수 있습니다.

```
기록 흐름:
AVFoundation 소음 기록 ──┐
HealthKit 보행 데이터   ──┼──▶ 4축(몸) 자동 연결 ──▶ 주간·월간 대시보드
복약 체크리스트          ──┘
```
