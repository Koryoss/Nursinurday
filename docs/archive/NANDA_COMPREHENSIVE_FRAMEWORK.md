# CareFlow NANDA 간호 진단 통합 프레임워크 (최종)

**목적**: 간호진단과 협력문제의 명확한 구분 + 객관적/주관적 자료 분석

---

## 🚦 핵심 구분: 간호진단 vs 협력문제 × 신호등

### 🟢🟡🟠 간호진단 (Nursing Diagnosis) — CareFlow 영역
```
🟢 Readiness for enhanced ~~ (안녕 간호진단)
   정의: 현재 건강하며 더 향상시키고 싶은 상태
   예시: "Readiness for enhanced stress management"
   CareFlow: 자기돌봄 강화 제안 / S 자료만 OK

🟡 Risk for ~~ (위험 간호진단)
   정의: 아직 발생하지 않았으나 위험 신호 있음
   예시: "Risk for anxiety related to sleep deprivation"
   CareFlow: 패턴 인식 + 예방 모니터링 / O+S 필수

🟠 Actual ~~ Related To (실제 간호진단)
   정의: 이미 발생한 심리적/신체적 반응
   예시: "Disturbed sleep pattern related to anxiety"
   CareFlow: 현상 기반 심리 지원 + 자기돌봄 / O+S 필수
```

### 🔴 협력문제 (Collaborative Problems) = Potential Complication
```
🔴 PC: [의료적 합병증] — 의료 전문가만 가능

정의: 의료가 진단하고 치료해야 하는 문제
범위: 신체 질환, 정신 질환, 합병증 위험
예시:
  🔴 PC: Hearing loss (이명 → 이비인후과)
  🔴 PC: Major depression (우울증 → 정신과)
  🔴 PC: Suicide attempt (자살 위험 → 1393 즉시)

CareFlow 역할: 🔴 신호 감지 → 즉시 의료 연결 → 심리적 지원
(진단/치료는 의료만 가능)
```

## 📊 자료 수집: 객관적(O) vs 주관적(S)

### 주관적 자료 (Subjective Data) — S
```
사용자가 직접 말하는 것
- "귀에서 계속 소리가 들려요" (이명 호소)
- "밤에 잠을 못 자요" (수면 문제)
- "스트레스가 많아요" (스트레스 인식)
- "불안해요", "무의미해요" (감정 표현)
- "뭔가 나를 감시하는 것 같아요" (지각)

특징: 개인의 경험, 느낌, 인식
신뢰도: 환자 자신의 진술이므로 중요하지만, 객관적 자료로 확인 필요
```

### 객관적 자료 (Objective Data) — O
```
관찰, 측정, 검사로 얻는 자료
- 센서 데이터: 심박수, 수면 패턴, 신체 움직임 (Apple Watch)
- 행동 관찰: 반응 속도, 활동 수준, 사회적 상호작용
- 신체 신호: 손떨림, 안색 변화, 목소리 톤
- 기능 수행: 일상 활동 (밥 먹기, 샤워, 외출) 여부
- 의료 검사: 혈액검사, 영상 검사 결과 (의료 영역)

특징: 객관적이고 측정 가능
신뢰도: 높음 (신뢰할 수 있는 관찰과 측정)
```

## 🔄 간호진단 도출 과정 (PES 형식)

### P-E-S 구조
```
P (Problem): 간호 진단명
E (Etiology): 원인/관련 요인 (자극)
S (Signs/Symptoms): 증거/증상 (객관적 + 주관적 자료)
```

### 🟠 예시 1: 불안 관련 실제 간호진단 (Actual Nursing Diagnosis)

**사용자 메시지**: "요즘 항상 불안하고, 밤에 잠을 못 자요. 심장이 자주 철렁거려요"

**자료 수집**:
```
주관적 자료(S):
  - "항상 불안해요" (불안 호소)
  - "밤에 잠을 못 자요" (수면 문제)
  - "심장이 철렁거려요" (신체 감각 호소)

객관적 자료(O):
  - Apple Watch: 야간 심박수 변동성 증가 (65→100 bpm)
  - 수면 기록: 자주 깸, 총 수면 4시간
  - 행동 관찰: 답변 속도 빠름, 손가락 움직임 빈번
```

**🟠 간호진단 도출** (Related To 형식):
```
P-E-S:
P: Disturbed sleep pattern
E: Related to anxiety and autonomic nervous system activation
S:
  주관: "항상 불안해요", "밤에 못 자요", "심장이 철렁거려요"
  객관: 야간 심박수 증가, 수면 부족, 빠른 반응 속도

🟠 NANDA 유형: 실제 간호진단 (Actual)
🔴 협력문제 여부: 없음 (아직 의료 진단 필요 없음)
    → 만약 2주 이상 지속되면 🔴 정신과 연결 고려
```

**CareFlow 응답** (v4.1 "Related To"):
```
1️⃣ 자료 인정:
   "불안과 관련된 수면 방해를 경험하고 계신 거군요.
    데이터로도 보이는데, 야간 심박수가 높고 수면이 단편화되어 있어요."

2️⃣ 🟠 현상 파악:
   "불안과 관련된 수면 방해"
   자극: 업무 스트레스 → 신경계 과활성화 → 야간 각성

3️⃣ 함께 관리하기:
   - 4-7-8 호흡법: 잠자기 전 실행
   - 침실 환경: 온도 낮추기, 소음 제거
   - 아침 햇빛: 매일 10분 (신체 시계 리셋)
   - 저녁 운동: 3시간 전 완료

4️⃣ 모니터링:
   🟡 "2주 후에도 계속되면 전문가 상담 권유"
   "수면 시간과 깨는 횟수를 2주간 기록해 주세요"
```

### 예시 2: 협력문제가 동반된 경우

**사용자 메시지**: "3개월 전부터 귀에서 자꾸 울리는데, 이때문에 밤에 못 자고 우울해져요. 이게 이명인가요?"

**자료 수집**:
```
주관적 자료(S):
  - "귀에서 계속 울려요" (이명 호소)
  - "밤에 못 자요" (수면 문제)
  - "우울해져요" (정서 변화)
  - "이게 이명인가요?" (진단 추론)

객관적 자료(O):
  - Apple Watch: 야간 심박수 불규칙, 수면 부족 3개월
  - 활동 수준: 외출 횟수 감소 (주 3회 → 주 1회)
  - 반응: 느린 답변 속도, 톤 낮음
  - 의료 데이터: (아직 의료 확인 없음)
```

**🔴 협력문제 감지** → 핵심!
```
❌ CareFlow가 할 수 없는 것:
   "당신은 이명입니다" (진단 불가)
   "내이 문제입니다" (원인 판단 불가)

✅ CareFlow가 해야 할 것:
   🔴 협력문제(PC) 인식 → 즉시 의료 연결
```

**🔴 도출 결과**:
```
🔴 협력문제: PC: Hearing loss / PC: Depression
   → 이비인후과 상담 즉시

🟠 간호진단 (동반 지원):
   "이명 증상과 관련된 수면 방해 및 우울감"
   (Related To 형식)
   - P: Disturbed sleep + Depressed mood
   - E: Related to tinnitus perception
   - S: 밤 울림 증폭, 3개월 수면 부족, 활동 감소
```

**CareFlow 응답** (v4.1 "Related To"):
```
1️⃣ 🔴 의료 연결 (최우선):
   "귀에서 울리는 소리가 3개월간 지속된다니,
    먼저 이비인후과에 가서 확인받는 게 정말 중요해요."

2️⃣ 🟠 현상 파악 (병행):
   "이명 증상과 관련된 수면 방해를 경험하고 계신 거군요.
    의료 확인을 기다리는 동안 함께 관리해 봐요."
   - 음향 마스킹 (백색소음)
   - 주의 전환 (책 읽기, 명상)
   - 수면 위생 개선

3️⃣ 모니터링:
   "이비인후과 진료 후 어떤 진단이 나오든,
    그 증상과 함께 생활하는 방법을 함께 배워갈 거예요"
```

## 🔴 협력문제의 4가지 유형 (Potential Complication)

### 1️⃣ 🔴 신체 증상 기반 협력문제

| 🔴 증상 | PC (Potential Complication) | 의료 영역 | CareFlow 역할 |
|--------|---------------------------|---------|------------|
| 이명 | PC: Hearing loss | 이비인후과 | 🔴 즉시 연결 + 🟠 심리 지원 |
| 어지럼증 | 낙상 위험 합병증 | 신경과/이비인후과 | 안전 가이드 + 심리 지원 |
| 비문증 | 안과 질환 여부 확인 | 안과 | 의료 연결 + 불안 관리 |
| 메니에르병 | 의료 진단 + 관리 | 이비인후과 | 의료 연결 + 복합 심리 지원 |

### 2️⃣ 정신건강 기반 협력문제

| 증상 | 협력문제 | 의료 영역 | CareFlow 역할 |
|------|---------|---------|------------|
| 우울증 증상 (2주+) | 우울증 진단 + 약물치료 | 정신과 | 의료 연결 + 심리 지원 |
| 불안장애 증상 | 불안장애 진단 + 치료 | 정신과 | 의료 연결 + 자기돌봄 |
| 공황장애 | 공황 진단 + 치료 | 정신과 | 응급 안정화 + 의료 연결 |
| 정신증 증상 (환청/환각) | 정신과 응급 진단 | 정신과 응급 | **1577-0199 즉시** |
| 자살 위험 | **생명 위협** | **응급실** | **1393 즉시** |

### 3️⃣ 약물 부작용 기반 협력문제

```
약물 복용 후 새로운 증상 발생
→ 협력문제: "약물 부작용 의심"
→ 의료 영역: 처방의와 상담
→ CareFlow 역할: 부작용 인식 + 의료 연결 + 심리 지원
```

### 4️⃣ 합병증 진행 기반 협력문제

```
기저 증상 → 악화 → 합병증 위험
예: 이명 (3개월 지속) → 청력 손상 위험 + 우울증 심화
→ 협력문제: "청력 손상 합병증 위험", "우울증 심화"
→ CareFlow 역할: 모니터링 + 의료 재상담 권유
```

## 📋 NANDA 4가지 유형 (간호진단)

| 유형 | 정의 | 자료 | CareFlow 역할 | 협력문제 |
|------|------|------|-------------|---------|
| 1️⃣ 안녕 진단 | 건강 + 향상 의지 | S만 필요 | 자기돌봄 강화 | 없음 |
| 2️⃣ 위험 진단 | 위험 신호 있음 | O + S로 위험 확인 | 예방 + 모니터링 | 없거나 모니터링 |
| 3️⃣ 실제 진단 | 현재 문제 발생 | O + S로 문제 확인 | 심리적 지원 | **동반 가능** |
| 4️⃣ (추가) 협력문제 | **의료 영역** | O + S로 의료 신호 확인 | **의료 연결** | **항상 존재** |

## ✅ 간호진단 vs 협력문제 판단 플로우

```
사용자 메시지 입력
  ↓
Step 1: 주관적 자료(S) 수집
  "귀에서 소리가 들려요", "불안해요"
  ↓
Step 2: 객관적 자료(O) 수집
  Apple Watch 센서, 활동 패턴, 행동 관찰
  ↓
Step 3: 협력문제 감지 확인
  ┌─ YES: 신체/정신 질환 진단 필요?
  │       ("이명인가요?", "우울증인가요?")
  │   → 협력문제! 의료 연결
  │
  └─ NO: 의료 진단 필요 없음
      → 간호진단 도출 (NANDA 분류)
      ↓
      Step 4: NANDA 간호진단 도출
      - 안녕 진단 (wellness)
      - 위험 진단 (risk)
      - 실제 진단 (actual)
      ↓
      Step 5: CareFlow 중재
      - 자기돌봄 강화/예방/심리 지원
```

## 🔍 실전 분석 예시 5가지

### Case 1: 순수 간호진단 (협력문제 없음)

**사용자**: "스트레스가 많아서 밤에 자주 깨요"

```
S: "스트레스 많아요", "밤에 자주 깨요"
O: Apple Watch - 야간 각성 2-3회, 심박수 불규칙
협력문제: 없음 (의료진단 필요 없음)

간호진단: "스트레스로 인한 수면 방해"
CareFlow: 자기돌봄 (호흡법, 수면 위생)
```

### Case 2: 협력문제 + 간호진단

**사용자**: "2주일 동안 아무것도 안 하고 싶고 밥맛도 없어요"

```
S: "아무것도 안 하고 싶어요", "밥맛 없어요"
O: 활동 감소, 식사량 감소, 느린 반응 속도
협력문제: YES! "우울증 진단 필요"

Step 1 - 의료 연결:
  "정신과나 정신건강의학과에 가세요"
Step 2 - 간호진단 (진료 전후):
  "무의미감으로 인한 활동 불내증"
Step 3 - CareFlow:
  의료 치료 + 심리 지원 병행
```

### Case 3: 협력문제만 있음

**사용자**: "3개월 전부터 귀에서 울리는데..."

```
S: "귀에서 울려요"
O: (의료검사 아직 안 됨)
협력문제: YES! "이명 진단 필요"

CareFlow:
- 의료 연결 우선
- 진단 후 심리 지원 시작
```

### Case 4: 위험 간호진단 (협력문제 경계)

**사용자**: "최근 수면이 불규칙해져서..."

```
S: "수면 불규칙해요"
O: 수면 패턴 감소 추세 (일주 감소)
협력문제: 아직 없지만 위험 신호 있음

간호진단: "수면 부족 위험 진단"
CareFlow:
- 예방적 중재 (수면 위생)
- 모니터링 (2주 후 추적)
- 악화 시 협력문제로 전환 가능
```

### Case 5: 긴급 협력문제

**사용자**: "죽고 싶어요"

```
S: "죽고 싶어요"
O: (객관적 자료 수집 불필요, 즉시 대응)
협력문제: **YES! 생명 위협**

CareFlow: AI 호출 없음
→ 1393 자살예방상담전화 즉시 안내
```

## 📌 최종 원칙

```
1️⃣ 간호진단 vs 협력문제
   간호진단: "심리적 반응" (CareFlow 담당)
   협력문제: "의료적 문제" (의료 담당)

2️⃣ 객관적(O) + 주관적(S) 자료
   둘 다 수집해서 정확한 판단
   S만으로는 협력문제 판단 곤란

3️⃣ 협력문제 → 의료 연결이 최우선
   진단은 의료만 가능
   CareFlow는 의료 후 심리 지원

4️⃣ 간호진단 → CareFlow 중재 가능
   자기돌봄, 예방, 심리적 지원
```
