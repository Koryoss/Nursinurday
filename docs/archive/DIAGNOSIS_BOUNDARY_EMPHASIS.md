# CareFlow 의료 진단 경계 강화 문서

**작성일**: 2026-03-23
**목적**: CareFlow의 핵심 원칙 "의학적 진단을 하지 않고 질병에 대한 인간의 반응에 집중" 강조

---

## 핵심 원칙 (Non-Negotiable)

```
CareFlow는 의학적 진단을 하지 않습니다.
그 질병에 대한 인간의 반응(심리적 영향, 불안감, 생활 제약)에만 집중합니다.
```

### 이 원칙이 적용되는 모든 것

| 영역 | 예시 |
|------|------|
| 신체 증상 | 이명, 비문증, 어지럼증, 메니에르병 |
| 정신건강 | 우울증, 불안장애, 공황장애, PTSD |
| 신경질환 | 편두통, 떨림, 마비 |
| 만성질환 | 당뇨, 고혈압, 자가면역질환 |
| 기타 | 수면장애, 식욕 부진, 피로 |

### 4. XCODE_SETUP_GUIDE.txt

#### 변경 내용

**파일 시작 부분**
- ⚠️ 중요 배너: 의료 진단이 아니라 신체 신호 관찰임을 명시

**⚡ 핵심 데이터 흐름 섹션**
- 제목 변경: "의료 진단이 아닌 신체 신호 관찰" 강조
- 파이프라인 각 단계에 "(의료진단 X)" 또는 "(관찰용)" 표기
- 목적 명확화: 심리적 영향 이해 + 자기돌봄 지원 + 전문의 상담 권유

**📂 파일 구조 요약**
- 처음에 ⚠️ 주의 배너 추가
- 각 파일 역할에 "(관찰용)", "(진단 X)" 표기
- MotionSensorManager, WatchDizzinessDetector에 ⭐ + "패턴 감지용 (의료진단 X)" 표기

**🚀 Step 6: CareFlow 웹 서버와 연동**
- 섹션 제목 변경: "신체신호 관찰 + 심리적 지원" 추가
- ⚠️ 배너: 의료 진단/의료 결정의 근거가 아님을 명시
- 대시보드 표시 내용에 "⚠️ 의료 진단이나 의료 조언은 포함되지 않습니다" 추가

**새로운 섹션: ⚠️ 법적/윤리적 경계**
- ❌ 금지 사항 명확히 나열
- ✅ 할 일 명확히 나열
- 각 항목 구체적 예시 제공

## 구현 가이드 (개발팀용)

### 시스템 프롬프트 작성 시

```
❌ 금지
"사용자는 불안장애를 가지고 있습니다"
"이명으로 진단됩니다"
"우울증 증상을 보입니다"

✅ 허용
"스트레스와 수면 부족이 당신의 불안감을 크게 증폭시키고 있네요"
"귀에 들리는 그 소리가 일상을 많이 방해하고 계신가요?"
"요즘 무의미감이 많이 느껴지시는군요"
```

### 신체증상 응답 템플릿

```
1️⃣ 의료 상담 우선: "[해당 전문과]에 가서 확인받는 게 좋을 것 같아요"

2️⃣ 심리적 영향 관찰: "그 증상으로 인해 [불안/무의미감/일상 제약]을 느끼고 계신 거군요"

3️⃣ 함께 관리하기:
   - 현재 상태에서 할 수 있는 자기돌봄
   - 증상과 함께 살아가는 방법
   - 지지 메시지

❌ 절대 하면 안 되는 것:
   - 의료 진단명 사용
   - 약물 조언
   - 원인 단정
   - 완치 약속
```

## 위기 상황 프로토콜 (반드시 준수)

### 🔴 Critical (자해/자살)
```
즉시 응답:
"정말 힘들 시겠군요. 지금 바로 전문가와 이야기하는 게 중요해요."
"자살예방상담전화 1393으로 전화해 주세요. 지금 바로요."
```

### 🟠 Urgent (심각한 증상)
```
"이런 증상이 있으시면 꼭 정신건강위기상담전화 1577-0199로 연락해 주세요."
"의료 전문가의 빠른 도움이 필요한 상황 같습니다."
```

## 법적 고지

- CareFlow는 의료기기가 아닙니다 (MDSAP, CE 마크 미인정)
- 의료 조언이 아닙니다
- 의료 진단이 아닙니다
- 의료 전문가를 대체하지 않습니다
- 응급 상황에서는 즉시 119 또는 응급실 방문
