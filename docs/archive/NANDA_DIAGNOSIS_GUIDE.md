# NANDA 간호 진단의 4가지 유형 — CareFlow 구현 가이드

**목적**: CareFlow가 사용자를 4가지 NANDA 유형으로 분류하고 각각 어떻게 응답하는지 명확히 하기

---

## 📋 NANDA 4가지 유형 개요

| 유형 | 상태 | CareFlow 역할 | 의료 전문가 개입 |
|------|------|-------------|---------------|
| 1️⃣ **안녕 간호진단** (Wellness) | 건강하고 더 향상 의지 | 자기돌봄 강화 | 선택적 |
| 2️⃣ **위험 간호진단** (Risk) | 문제 위험 신호 있음 | 예방 + 모니터링 | 모니터링 |
| 3️⃣ **실제 간호진단** (Actual) | 현재 심리적 어려움 | 지원 + 자기돌봄 | 권장 |
| 4️⃣ **협력문제** (Collaborative) | **의료 개입 필수** | **의료 연결 + 심리 지원** | **필수** |

---

## 1️⃣ 안녕 간호진단 (Wellness Diagnosis)

### 정의
- 사용자가 **현재 건강한 상태**
- **더 발전하고 싶은 의지**를 명확히 보임
- NANDA: "Readiness for Enhanced [domain]"

### 감지 키워드
```
"최근 괜찮은데 더 나아지고 싶어요"
"스트레스 관리를 배우고 싶어요"
"수면 습관을 개선하고 싶습니다"
"명상을 시작해 보고 싶어요"
```

### CareFlow 응답 예시

**사용자**: "최근 스트레스가 많이 줄었는데, 더 효과적으로 관리하는 방법을 배우고 싶어요"

```
✅ CareFlow:
1️⃣ 인정: "정말 좋은 진전이에요! 지금의 변화를 유지하면서 더 깊이 있게 가고 싶으신 거네요."

2️⃣ 현재 강점 확인: "지금까지 어떤 방법들이 가장 효과적이었나요?"

3️⃣ 자기돌봄 강화 제안:
   - 명상/요가 등 신체 기반 접근
   - 정서 인식 높이기 (마음챙김)
   - 일상 속 의미 찾기

4️⃣ 지속 격려: "이 흐름을 계속 이어가시는 거 정말 멋져요!"
```

### 코드 로직
```typescript
if (nandaDiagnosisType === 'wellness') {
  // 자기돌봄 강화 권고만 제공
  return buildEnhancementResponse(assessment)
  // AI: "더 발전하고 싶으신 마음을 함께 응원합니다"
}
```

## 2️⃣ 위험 간호진단 (Risk Nursing Diagnosis)

### 정의
- **문제가 아직 발생하지 않았으나** 위험 신호가 있음
- 추가 자료 수집이 필요할 수 있음
- NANDA: "Risk for [problem]"

### 감지 키워드
```
"최근 수면이 좀 불규칙해져서..." (수면 부족 → 불안 위험)
"요즘 혼자라는 생각이 들어요" (고립 위험)
"스트레스가 쌓이는 느낌이 들어요" (소진 위험)
"일에 집중이 안 되고 있어요" (우울 위험)
```

### CareFlow 응답 예시

**사용자**: "최근 회사 일이 많아져서 밤 11시, 12시까지 자는데, 이대로 괜찮을까요?"

```
✅ CareFlow (위험 신호 감지):
1️⃣ 신호 인정: "수면 부족이 계속되면 신체와 마음에 영향을 줄 수 있어요."

2️⃣ 예방 조언: "지금부터 작은 것부터 시작해 보세요."
   - 매일 8시간 수면 목표 설정
   - 자기 30분 전 스크린 끄기
   - 기상 시간을 일정하게 유지

3️⃣ 모니터링: "앞으로 2주간 수면 패턴을 기록해 보세요."
   - 밤에 깼을 때 시간, 깬 이유
   - 아침 피로감 (1-10 점수)
   - 낮 집중력 변화

4️⃣ 추후 지원: "만약 수면이 3주 이상 개선되지 않으면 말씀해 주세요."
```

### 코드 로직
```typescript
if (nandaDiagnosisType === 'risk') {
  // 위험 신호 감지 + 예방 자기돌봄
  const preventionStrategy = buildPreventionResponse(assessment)
  const monitoringPlan = generateMonitoringPlan(assessment)

  return {
    message: preventionStrategy + monitoringPlan,
    followUpNeeded: true,
    reviewAfterDays: 14
  }
}
```

## 3️⃣ 실제 간호진단 (Actual Health Problem)

### 정의
- **현재 심리적 어려움이 발생** 중
- 사용자가 직접 겪고 있는 문제
- NANDA: "[Problem] related to [cause] as evidenced by [signs/symptoms]"

### 감지 키워드
```
"너무 불안해서 잠을 못 자요"
"공허감이 계속되고 있어요"
"혼자라는 생각에 외로워요"
"의욕이 없어요"
```

### CareFlow 응답 예시

**사용자**: "최근 3주간 밤마다 불안해서 잠을 제대로 못 자고 있어요. 무섭기도 하고..."

```
✅ CareFlow (실제 간호진단):
1️⃣ 현재 상태 인정: "3주간 매일 그 불안을 견디고 계셨군요. 정말 힘드셨을 것 같아요."

2️⃣ NANDA 관찰 (내부용):
   진단: "불안과 수면 패턴 장애"
   자극: 업무 스트레스 + 새로운 환경
   경로: 신경계 과활성화 + 불안 악순환
   반응: 밤 불안 + 수면 단절 + 주간 피로
   정서: 두려움, 무력감, 내일이 올까 봐 두려움

3️⃣ 함께 관리하기:
   의료 확인: "이비인후과나 신경과 진료도 고려해 보세요"
   현재 실천:
   - 저녁 따뜻한 차, 복식호흡
   - 잠자리에서 불안 대신 신체 감각에 집중
   - 아침 햇빛 10분 (신체 시계 리셋)

4️⃣ 심리적 지원: "그 불안이 약해질 때까지 함께할 거예요."

5️⃣ 추후 확인: "1주 후 상태 어떤지 말씀해 주세요."
```

### 코드 로직
```typescript
if (nandaDiagnosisType === 'actual') {
  // NANDA 6단계: 진단→자극→경로→반응→정서→함께 관리하기
  const response = buildNANDAResponse({
    observation: assessment.primaryDomain,
    stimulus: extractTriggers(userMessage),
    pathophysiology: generateMechanism(assessment),
    response: extractSymptoms(userMessage),
    emotion: assessEmotionalState(assessment),
    therapeutic: generateSelfCareInterventions(assessment)
  })

  return response
}
```

## 4️⃣ 협력문제 (Collaborative Problems) = Potential Complication (합병증) — ⚠️ CareFlow의 경계

### 정의
- **의료 전문가의 개입이 필수적인 문제**
- Potential Complication (잠재적 합병증): 현재 증상에서 발생할 수 있는 의료적 위험
- CareFlow의 자기돌봄 범위를 **벗어남**
- CareFlow의 역할: 의료 연결 + 심리 지원 (진단/치료 대체 절대 금지)

### NANDA 협력문제의 분류

```
협력문제 = 간호 진단의 범위를 벗어나는 의료적 문제

1️⃣ 합병증(Complication of disease)
   - 기존 질환으로부터 발생하는 추가적 건강 문제
   - 예: 어지럼증 → 낙상 위험 → 골절 합병증
   - 예: 우울증 → 자살 시도 합병증

2️⃣ 약물 부작용(Medication reaction)
   - 약물 복용으로 인한 의료적 문제
   - CareFlow 범위: 약물 부작용 증상으로 인한 심리적 영향만 다룸

3️⃣ 의료 시술 관련(Procedure-related)
   - 의료 검사나 치료로 인한 합병증
   - CareFlow 범위: 시술 후 불안이나 회복 심리 지원

4️⃣ 진행성 질환(Disease progression)
   - 증상 악화 또는 새로운 증상 출현
   - CareFlow 범위: 악화로 인한 심리적 영향 지원
```

### 협력문제 감지 기준

#### 신체증상 기반 협력문제
```
❌ CareFlow가 할 수 없는 것 (의료 필수)
"이명이 들려요"           → "이명입니다" 진단 금지
"눈앞에 뭔가 떠요"        → "비문증입니다" 진단 금지
"어지럼증이 있어요"       → "어지럼증입니다" 진단 금지
"메니에르병인 것 같아요"  → "메니에르병입니다" 진단 금지

✅ CareFlow가 할 수 있는 것
"귀에 울림이 들린다니 얼마나 신경 쓰이세요?"
"먼저 이비인후과에 가서 확인받는 게 좋을 것 같아요"
"그 증상으로 인한 불안을 함께 살펴봐요"
```

#### 정신건강 기반 협력문제
```
❌ 심각한 정신 증상 (의료 필수)
"자살 생각이 들어요"     → 1393 즉시
"환청이 들려요"         → 1577-0199 즉시
"극도의 공황 상태"      → 응급실 권유
"자해 충동"             → 1393 즉시

✅ 의료 상담 권유 (경고하지 말고 지원)
"정말 힘든 상태이신 것 같아요"
"정신건강위기상담전화 1577-0199로 전화하시면 좋을 것 같아요"
"의료진이 도움을 줄 수 있을 거예요"
```

### CareFlow 응답 예시

#### 신체증상 기반 협력문제 (Medical Health Awareness)

**사용자**: "2개월 전부터 귀에서 계속 윙윙거리는 소리가 들려요. 밤에 심해져서 잠을 못 자요"

```
✅ CareFlow (협력문제 감지):

1️⃣ 의료 연결 (최우선):
   "귀에서 울리는 그 소리, 정말 신경 쓰이시겠네요.
    먼저 이비인후과에 가셔서 확인받는 게 꼭 필요할 것 같아요.
    [병원 찾기 팁 제공]"

2️⃣ 의료 대기 중 심리적 지원:
   "이비인후과 진료를 기다리면서,
    그 소리로 인한 불안과 수면 방해를 함께 살펴봐요."

3️⃣ NANDA 응답 (협력문제 + 심리적 영향):
   진단: "이명 지각으로 인한 수면 방해 + 불안"
   자극: 내이 불균형(?), 스트레스 누적
   경로: 청각신경 과민화 + 주의 편향 + 수면 악순환
   반응: 밤 울림 증폭 → 잠 못 잠 → 피로 → 더 신경 쓰임
   정서: "이게 평생 계속되면?" → 불안, 무의미감, 절망감

4️⃣ 현재 할 수 있는 자기돌봄:
   - 음향 마스킹 (백색소음, 자연음)
   - 주의 전환 기술 (책 읽기, 명상)
   - 수면 위생 개선
   - 스트레스 감소 (운동, 요가)

5️⃣ 진료 후 지원:
   "진료 후 어떤 진단이 나오든,
    우리는 그 증상과 함께 일상을 어떻게 살아갈지 함께 생각해 봐요."
```

#### 정신건강 기반 협력문제 (Mental Health Awareness)

##### 💡 정신건강 협력문제의 3가지 수준

| 수준 | 상황 | CareFlow 대응 | 의료 개입 |
|------|------|-------------|---------|
| 🔴 **긴급** | 자살/자해 직접 위험 | 1393 즉시 안내 | 응급실 |
| 🟠 **심각** | 환청/환각, 극도 기능저하 | 1577-0199 즉시 | 정신과 응급 |
| 🟡 **주의** | 우울증, 불안장애, 공황 증상 | 정신건강 센터 권유 | 정신과 상담 |

##### 🔴 **긴급 협력문제**: 자살/자해 위험

**사용자**: "계속 죽으면 좋겠다는 생각이 들어요"

```
✅ CareFlow (즉시 대응):

1️⃣ 공감 (판단 없음):
   "그렇게 힘든 마음이 들고 계시는군요.
    그런 생각이 드신 건 당신이 약해서가 아니라
    정말 힘든 상황 때문일 거예요."

2️⃣ 즉시 의료 연결 (AI 호출 없음):
   "지금 바로 자살예방상담전화 1393으로 전화해 주세요.
    24시간 전문가가 당신의 이야기를 들어줄 거예요.

    또는 가장 가까운 응급실로 가세요.
    당신을 도울 전문가들이 있어요."

3️⃣ 현재 안전화:
   - 위험한 물건 멀리하기
   - 신뢰할 수 있는 사람 곁에 있기
   - 자살 시도 계획이 있다면 반드시 알려주기

4️⃣ AI 호출 절대 금지:
   ❌ 이 경우는 심리 상담이 아니라 의료 응급
   ❌ "당신은 충분히 소중해요" 같은 위로 금지
   ❌ 문제 해결 제안 금지
```

##### 🟠 **심각 협력문제**: 심각한 정신증상

**사용자**: "며칠 전부터 누군가 나를 감시한다는 생각이 계속 들어요. 귀에서 목소리도 들려요"

```
✅ CareFlow (긴급 의료 연결):

1️⃣ 공감 + 신호 인정:
   "그런 경험이 정말 두렵고 혼란스럽겠네요.
    그런 증상들은 의료 전문가의 도움이 정말 필요해요."

2️⃣ 즉시 의료 안내:
   "정신건강위기상담전화 1577-0199로 전화하세요.
    또는 가까운 정신과나 응급실에 가시길 권해요.

    증상:
    - 환청/환각
    - 편집증적 사고
    - 현실 감각 손상

    이런 것들은 의학적 치료가 필요합니다."

3️⃣ 보호자/신뢰인 연결:
   "보호자나 신뢰하는 사람과 함께 병원에 가시는 게 좋아요."

4️⃣ CareFlow의 역할 제한:
   ❌ "이건 정신분열증입니다" 진단 금지
   ❌ 약물 제안 금지
   ✅ "의료 전문가가 도와줄 수 있어요" 지지만
```

##### 🟡 **주의 협력문제**: 정신건강 장애 (의료 상담 필수)

**예시 1: 우울증 협력문제**

**사용자**: "2주일 동안 아무것도 재미가 없고, 자꾸 울고 싶고, 밥을 먹어도 맛이 없어요"

```
✅ CareFlow (의료 상담 권유 + 심리 지원):

1️⃣ 현재 상태 인정:
   "2주일을 그렇게 버티셨군요. 정말 힘드셨을 거예요.
    그런 증상들은 의료 확인이 필요합니다."

2️⃣ 의료 상담 권유:
   "정신과나 정신건강의학과에 가서 전문가와 이야기 나누세요.
    혼자가 아니고, 도움을 받을 수 있어요."

3️⃣ 현재 상태에서의 심리적 지원 (의료 진료 전후):
   NANDA 3️⃣ 실제 간호진단으로 전환:

   진단: "우울감으로 인한 무력감 및 흥미 상실"
   자극: 최근 스트레스 사건 or 생리적 변화
   경로: 신경생리적 변화 + 부정적 사고 악순환
   반응: 에너지 저하, 식욕 부진, 울음, 무관심
```

## 🎯 실제 구현 플로우 (pseudocode)

```typescript
async function analyzeUserMessage(message: string) {
  // 1단계: NANDA 진단 유형 분류
  const nandaType = assessNANDAType(message)
  // 'wellness' | 'risk' | 'actual' | 'collaborative'

  // 2단계: 위기 수준 동시 판단
  const crisisLevel = detectCrisisLevel(message)
  // 'critical' | 'urgent' | 'monitor' | 'none'

  // 3단계: NANDA 유형에 따른 응답 생성
  if (crisisLevel === 'critical') {
    // ⚠️ 가장 우선: 생명 위기 응답
    return getCrisisResponse()
  } else if (nandaType === 'collaborative') {
    // 협력문제: 의료 연결 + 심리 지원
    return {
      medical: getReferralInfo(assessment.domain),
      psychological: buildSupportWithReferral(assessment)
    }
  } else if (nandaType === 'wellness') {
    // 안녕 간호진단: 강화
    return buildEnhancementResponse(assessment)
  } else if (nandaType === 'risk') {
    // 위험 간호진단: 예방 + 모니터링
    return buildPreventionResponse(assessment)
  } else {
    // 실제 간호진단: 지원
    return buildNANDAResponse(assessment)
  }
}
```

## 마지막: 협력문제의 의미

**협력문제 = CareFlow와 의료의 협력**

```
사용자 어려움
  ↙         ↘
의료팀      CareFlow
(진단/치료)  (심리/자기돌봄)
  ↖         ↗
    함께 지원
```

- CareFlow는 의료팀을 **대체하지 않고 보완**
- 의료팀은 신체/의학적 문제를 **담당**
- CareFlow는 심리적 영향을 **담당**
- 사용자는 **두 팀의 지원을 모두 받음**
