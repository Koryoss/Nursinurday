### 핵심 임상적 관찰 (v4.1 언어)

이 케이스의 본질은 **3개 감각 시스템이 동시에 만성 변화 상태**에 있다는 것이다.

```
전정계 (Vestibular)  →  균형감각 변화 + 어지러움 패턴
청각계 (Auditory)    →  지속적 이명 + 좌측 청력 변화
시각계 (Visual)      →  지속적 비문증
         ↓
   일상생활 전반에 누적 감각 부하 (Sensory Load)
```

메니에르증후군의 고전적 3요소(어지럼 + 이명 + 청력저하)가 **모두 존재**하며,
비문증은 독립 현상이나 전반적인 감각 과부하 상태를 심화시킨다.

## 3. CareFlow 모니터링 설계

### 3-1. O (Objective) 데이터 — Apple Watch 자동 수집

| 센서 | 측정 항목 | NANDA 연결 |
|---|---|---|
| 자이로스코프 | 회전 각속도 (rad/s) | A-1 균형감각 변화, PC-2 급성 발작 |
| 가속도계 | 기울기 변화 (°) | A-1, R-1 낙상 위험 |
| HealthKit 심박 | 심박수 + HRV | PC-3 약물 부작용, A-5 피로 |
| HealthKit 수면 | 수면 시간 + 질 | R-4 수면 교란 위험 |
| HealthKit 활동 | 걸음수 + 에너지 | R-2 활동 위축 위험, A-6 활동 변화 |

### 3-2. S (Subjective) 데이터 — 앱 내 자가 보고 (★ 신규 기능 필요)

현재 CareFlow는 O 데이터만 수집. 이 케이스를 위해 **자가 보고 화면** 추가가 필요하다.

| 항목 | 측정 방식 | 주기 | NANDA 연결 |
|---|---|---|---|
| 이명 강도 | 0~10 슬라이더 | 매일 아침 | A-2 청각 지각 변화 |
| 비문증 정도 | 0~10 슬라이더 | 매일 아침 | A-4 시각 지각 변화 |
| 어지러움 주관 강도 | 0~10 슬라이더 | 에피소드 후 | A-1 균형감각 변화 |
| 수면 만족도 | 0~10 슬라이더 | 매일 아침 | R-4 수면 교란 |
| 오늘의 컨디션 | 3단계 선택 (좋음/보통/힘듦) | 매일 아침 | A-5 만성 피로 |
| 병원 방문 기록 | 날짜 + 메모 | 방문 시 | 의료 연속성 |

### 3-3. 🚦 신호등 기준 — 이 케이스 맞춤 조정 제안

기본 임계값(20/50/80)은 일반 사용자 기준. 메니에르 환자는 **기저 신뢰도가 높게 유지**될 수 있어 조정이 필요할 수 있다.

| 구간 | 일반 | 이 케이스 제안 | 이유 |
|---|---|---|---|
| 🟢 none | < 20 | < 15 | 만성 상태로 기저 움직임 변화 상존 |
| 🟡 mild | 20~49 | 15~44 | 조기 감지 민감도 높임 |
| 🟠 moderate | 50~79 | 45~74 | |
| 🔴 severe | ≥ 80 | ≥ 75 | 발작 위험 조기 포착 |

> ⚠️ 실기기 테스트 후 실제 데이터 기반으로 재조정 필수

## 4. 간호 중재 방향 (v4.1)

### 각 신호등 단계별 대응

```
🟢 정상 모니터링
   → "오늘도 균형 상태가 안정적입니다"
   → 아침 자가 보고 알림
   → 이명/비문증 일지 기록 유도

🟡 주의
   → "균형감각 변화 가능성이 감지됩니다"
   → "천천히 움직이시고 벽이나 가구를 잡아주세요"
   → 활동 기록 자동 저장

🟠 감지됨
   → "전정기능과 관련된 균형 변화가 관찰됩니다"
   → "지금 앉거나 누워주세요"
   → 에피소드 자동 기록 시작
   → 지속 시간 측정

🔴 즉각 대응
   → "PC: 즉각적인 안정이 필요합니다"
   → "가까운 벽이나 바닥에 앉아주세요"
   → 보호자 알림 (★ 신규 기능 필요)
   → 에피소드 상세 기록 → 다음 외래 시 의사에게 제출
```

## 5. 2개월 외래 주기와 CareFlow 연동

이 케이스의 핵심 강점 중 하나는 **이미 의료 추적 관리가 되고 있다**는 것이다.
CareFlow는 진단 도구가 아니라 **"외래와 외래 사이 60일을 채우는 도구"** 다.

### 외래 전 요약 리포트 (★ 신규 기능 필요)

```
[지난 60일 요약]
• 어지러움 에피소드: 총 N회
  - 평균 지속시간: XX분
  - 최대 강도(confidence): XX
  - 주로 발생 시간대: 오전/오후/야간
• 이명 강도 평균: X.X / 10
• 수면 평균: X.X시간
• 활동량 추이: 증가/감소/유지
• 주관 컨디션 패턴: [그래프]
• 복용약 후 변화 패턴: [있음/없음]
```

→ 이 리포트를 PDF로 출력해서 의사에게 직접 제출 가능하게 하면,
환자가 "기억"에 의존하지 않고 **데이터로 의사와 대화**할 수 있다.

## 6. 이 케이스에서 CareFlow의 한계 (솔직하게)

| 한계 | 이유 | 대안 |
|---|---|---|
| 이명 직접 측정 불가 | Apple Watch에 청각 센서 없음 | 자가 보고 + 기록으로 보완 |
| 비문증 직접 측정 불가 | 안구 센서 없음 | 자가 보고로만 관리 |
| 약물 효과 직접 평가 불가 | 혈중 농도 측정 불가 | 에피소드 빈도·강도 간접 지표로 활용 |
| 좌우 청력 분리 측정 불가 | Watch는 청력계 아님 | S 데이터(주관 보고)로 보완 |
| 발작 예측 불가 | 사후 감지만 가능 | 패턴 학습 후 위험 시간대 알림 (미래 기능) |
